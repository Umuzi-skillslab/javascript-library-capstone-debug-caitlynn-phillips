========================

across three files (library.js, ui.js and library.test.js). Errors are 
categorized below by severity and JavaScript concept.

FIX STRATEGY
-------------
Fixes were applied in the following systematic order:

1. Variable declarations - establish correct scoping foundation
2. Class inheritance - fix OOP structure before testing any functionality
3. Comparison operators - fix all = to === errors
4. Control flow - fix infinite loop and recursive stack overflow
5. Functions - replace loops with array methods, add validation
6. ES6 modernisation - template literals, destructuring, spread, rest
7. DOM fixes - selectors, event listeners, event delegation
8. JSON & localStorage - stringify, parse, try/catch
9. Error handling - try/catch and null checks throughout
10. Test suite - write 15+ comprehensive tests with edge cases
